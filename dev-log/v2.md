## Token 超时机制

### 机制说明
完全基于数据库实现，不依赖 Redis：

1. **Token 生成**
   - 创建时设置 `createdAt` 和 `lastUsedTime`
   - `expired = false`

2. **Token 验证**
   - 检查 `expired` 字段
   - 计算当前时间与 `lastUsedTime` 的差值
   - 超过3分钟则标记为过期并拒绝访问

3. **玩家退出**
   - 更新 `lastUsedTime` 为当前时间
   - 不立即标记 `expired = true`
   - 允许3分钟内使用同一Token重新登录

4. **Token 续期**
   - 每次成功验证时更新 `lastUsedTime`
   - 重置3分钟超时计时器

5. **清理过期Token**
   - 定时任务检查 `lastUsedTime` 超过3分钟的Token
   - 标记为 `expired = true`
