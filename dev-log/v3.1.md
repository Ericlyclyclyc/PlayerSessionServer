## 需求概述
在v3.0的基础上，新增玩家删除功能，允许管理员删除玩家账户及其所有关联数据。

## 功能需求
1. **删除玩家账户**
   - 删除玩家时，级联删除所有关联的AccessToken和TokenEvent
   - 在线玩家无法删除，必须先退出游戏
   - 提供REST API端点供调用

2. **数据完整性**
   - 确保删除操作的安全性
   - 防止误删除在线玩家
   - 删除操作不可逆

### 2. Entity级联删除配置
**文件**: `src/main/java/org/lyc122/dev/playersessionserver/entity/AccessToken.java`

修改`player`字段，添加级联删除配置：
```java
@ManyToOne(fetch = FetchType.LAZY, cascade = CascadeType.ALL, orphanRemoval = true)
@JoinColumn(name = "player_id", nullable = false)
private Player player;
```

**文件**: `src/main/java/org/lyc122/dev/playersessionserver/entity/TokenEvent.java`

修改`accessToken`字段，添加级联删除配置：
```java
@ManyToOne(fetch = FetchType.LAZY, cascade = CascadeType.ALL, orphanRemoval = true)
@JoinColumn(name = "token_id", nullable = false)
private AccessToken accessToken;
```

**说明**:
- `CascadeType.ALL`: 删除Player时自动删除关联的AccessToken
- `orphanRemoval = true`: 删除Player时自动删除关联的AccessToken，删除AccessToken时自动删除关联的TokenEvent

### 3. Service层实现
**文件**: `src/main/java/org/lyc122/dev/playersessionserver/service/PlayerService.java`

新增`deletePlayer`方法：
```java
@Transactional
public void deletePlayer(String username, String uuid) {
    // 验证玩家存在
    Player player = validatePlayer(username, uuid);

    // 检查玩家是否在线
    if (player.getStatus() == Player.PlayerStatus.ONLINE) {
        throw new PlayerConflictException("无法删除在线玩家，请先退出游戏");
    }

    // 删除玩家（级联删除关联的AccessToken和TokenEvent）
    playerRepository.delete(player);
}
```

**逻辑流程**:
1. 验证玩家存在（用户名和UUID匹配）
2. 检查玩家是否在线
3. 如果在线，抛出`PlayerConflictException`异常
4. 如果离线，删除玩家（JPA自动处理级联删除）

### 4. Controller层实现
**文件**: `src/main/java/org/lyc122/dev/playersessionserver/controller/PlayerController.java`

新增`deletePlayer`端点：
```java
@DeleteMapping("/delete")
public ResponseEntity<ApiResponse<Void>> deletePlayer(@Valid @RequestBody DeletePlayerRequest request) {
    log.info("Delete player request for username: {}", request.getUsername());
    playerService.deletePlayer(request.getUsername(), request.getUuid());
    return ResponseEntity.ok(ApiResponse.success("玩家删除成功", null));
}
```

**端点信息**:
- 方法: `DELETE`
- 路径: `/api/players/delete`
- 认证: 需要API Key认证（生产环境）
- 参数验证: 使用`@Valid`注解验证请求参数

## 技术亮点

### 1. 级联删除优化
使用JPA的`CascadeType.ALL`和`orphanRemoval`特性，自动处理关联数据的删除，避免手动编写删除逻辑。

### 2. 安全性保障
- 在线玩家无法删除，防止误操作
- 双重验证（用户名和UUID），确保删除正确的玩家
- 使用事务（`@Transactional`）保证数据一致性

### 3. 错误处理
- 使用`PlayerConflictException`处理在线玩家删除冲突
- 使用`PlayerNotFoundException`处理玩家不存在的情况
- 统一的异常处理机制

## v3.1改进

### 优化删除玩家参数要求
**改进日期**: 2026-04-02

**改进内容**:
- 删除玩家功能不再要求同时提供username和uuid
- 支持只提供username或只提供uuid来识别玩家
- 如果同时提供username和uuid，会验证它们是否匹配

**修改文件**:
1. `DeletePlayerRequest.java` - 移除@NotBlank验证注解
2. `PlayerService.java` - 更新deletePlayer方法逻辑

**实现逻辑**:
```java
@Transactional
public void deletePlayer(String username, String uuid) {
    // 检查是否至少提供了一个标识符
    if ((username == null || username.trim().isEmpty()) && (uuid == null || uuid.trim().isEmpty())) {
        throw new InvalidCredentialsException("至少需要提供用户名或UUID中的一个");
    }

    Player player = null;

    // 根据提供的参数查找玩家
    if (username != null && !username.trim().isEmpty()) {
        if (uuid != null && !uuid.trim().isEmpty()) {
            // 同时提供了username和uuid，需要验证它们匹配
            player = validatePlayer(username, uuid);
        } else {
            // 只提供了username
            player = getPlayerByUsername(username);
        }
    } else {
        // 只提供了uuid
        player = getPlayerByUuid(uuid);
    }

    // 检查玩家是否在线并删除
    // ...
}
```

**用户体验改进**:
- 更灵活的参数选择，用户可以根据手中拥有的信息选择使用username或uuid
- 减少了必须同时提供两个参数的限制，降低了使用复杂度
- 保持了安全性，如果同时提供两个参数会验证匹配性

**测试用例更新**:
- 添加只使用username删除玩家的测试
- 添加只使用uuid删除玩家的测试
- 验证同时提供username和uuid的匹配验证
