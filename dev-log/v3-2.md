- **主要变更**: 添加Token失效功能规范

## 功能需求更新

### 新增功能：Token失效
- **API端点**: POST /api/tokens/revoke
- **用途**: 手动使token失效，用于安全措施
- **请求参数**: username, uuid, token
- **响应**: 成功返回200，失败返回适当错误码
- **行为**: 
  - 验证玩家身份（用户名和UUID匹配）
  - 验证token存在
  - 验证token属于该玩家
  - 使token失效（标记为过期）
  - 如果玩家在线，自动清理会话
  - 记录REVOKED事件

## 代码实现变更

### 1. 新增文件

#### RevokeTokenRequest.java
- **路径**: src/main/java/org/lyc122/dev/playersessionserver/dto/RevokeTokenRequest.java
- **功能**: Token失效请求DTO
- **字段**:
  - username: 玩家名（必填）
  - uuid: 玩家UUID（必填）
  - token: 要失效的token（必填）

### 2. 修改文件

#### TokenController.java
- **新增方法**: revoke()
- **功能**: 处理token失效请求
- **实现逻辑**:
  1. 验证玩家存在（用户名和UUID匹配）
  2. 验证token存在
  3. 验证token属于该玩家
  4. 调用tokenService.revokeToken()使token失效
  5. 如果玩家在线，调用sessionService.endSession()清理会话
  6. 返回成功响应

#### TokenService.java
- **新增方法**: 
  - revokeToken(String token): 使指定token失效（使用REVOKED事件）
  - getAccessTokenByToken(String token): 获取token（不验证有效性）
- **实现逻辑**:
  - revokeToken: 标记token为过期，记录REVOKED事件
  - getAccessTokenByToken: 从数据库查询token，不进行有效性验证

#### TokenEvent.java
- **新增枚举值**: REVOKED
- **功能**: 区分手动失效和自动过期
- **用途**: 记录在token历史中，用于审计和安全分析

#### SecurityConfig.java
- **新增端点**: /api/tokens/revoke
- **访问权限**: 临时允许（测试用）
- **生产建议**: 需要API Key认证

### 3. 数据库变更

#### token_events表结构更新
- **变更类型**: 修改枚举类型
- **变更内容**: 在event_type枚举中添加REVOKED值
- **执行SQL**: 
  ```sql
  ALTER TABLE token_events 
  MODIFY COLUMN event_type 
  ENUM('EXPIRED','GENERATED','INVALIDATED','RENEWED','VALIDATED','REVOKED') 
  NOT NULL;
  ```

## 问题记录与解决

### 数据库结构问题
- **问题**: token_events表的event_type枚举缺少REVOKED值
- **症状**: 首次revoke操作返回500错误
- **原因**: 新增的REVOKED事件类型未在数据库枚举中定义
- **解决**: 手动执行SQL更新数据库表结构
- **影响**: 中等（需要在生产部署时包含此变更）
- **预防**: 在部署脚本中包含数据库迁移步骤
